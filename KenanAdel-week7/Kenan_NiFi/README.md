# Real-Time Data Ingestion & Processing with Apache NiFi

## Project Description
This project demonstrates a comprehensive, real-time data ingestion and processing pipeline using a custom Python data generator and an Apache NiFi flow. The pipeline collects data from multiple disparate sources—both from local synthetically generated files and dynamically pulled from relational databases and processes the streams, and subsequently stores the finalized, unified data into a Hadoop Distributed File System (HDFS).

The Python script autonomously generates continuous streams of highly complex, "messy" JSON data (simulating missing values, nulls, and exact intra-file duplicates). The Apache NiFi template is configured to ingest this data alongside database tables, perform schema validation, remove duplicates, execute SQL-like queries on the stream flow, and reliably land the results in Hadoop.

## Architecture and Processor Overview
The NiFi template employs a robust sequence of processors to handle the data:
1. **ListFile & FetchFile:** Scans the designated local directory for the synthetic JSON files generated by Python and pulls their contents into NiFi flowfiles.
2. **QueryDatabaseTableRecord:** Extracts incrementally updated data directly from a relational database and brings it into the pipeline for unified processing alongside the file-based stream.
3. **ValidateRecord:** Uses schema-based validation to ensure all incoming records conform to the expected format. Valid data proceeds, and malformed files are appropriately routed.
4. **DeduplicateRecord:** Cleans the stream by keeping track of incoming IDs and successfully dropping the intentional duplicates generated by our Python script (or present in the DB).
5. **QueryRecord:** Performs inline data transformation and filtering using SQL standards directly on the flowing records before saving.
6. **PutHDFS:** Writes the cleaned, deduped, and structured datasets to Hadoop (HDFS) for long-term storage and analytical availability.
7. **LogAttribute:** Extensively placed throughout the flow points to log flowfile properties dynamically for debugging and provenance monitoring during Transformation Layers.

## Prerequisites and Required Dependencies

To run this project natively, you will need the following initialized:
- **Python 3.x**: To run the synthetic data generation script.
- **Apache NiFi (v2+ recommended)**: To host the ingestion framework.
- **Relational Database**: For configuring the DBCPConnectionPool used by the database retrieval node (e.g., PostgreSQL).
- **Hadoop (HDFS)**: Target directory setup and accessible core-site.xml/hdfs-site.xml configuration files so `PutHDFS` can resolve the filesystem.
- Write permissions for `D:\nifi_input\data` (or modify `generate.py` accordingly).

No external Python dependencies (`pip install`) are required outside of standard library modules (`os`, `json`, `time`, `random`, `datetime`).

## Step-by-Step Setup and Installation

### 1. Data Generator Setup
1. Clone this repository to your local machine.
2. Open `Generate_code/generate.py`. By default, `OUTPUT_DIR` is set to `D:\nifi_input\data`. Ensure this path is available or edit the script to an accessible path.
3. Keep this terminal handy to generate data when the flow is ready.

### 2. Apache NiFi Setup
1. Open your Apache NiFi web interface (`http://localhost:8080/nifi`).
2. **Import the NiFi Template**:
   - Locate the **Operate** palette on the left pane.
   - Click the **Upload Template** icon.
   - Select `NiFi_Template/Real-Time_Data_Ingestion_&_Processing (1).json` and click **Upload**.
3. **Instantiate the Template**:
   - Drag the **Template** module from the top toolbar to the canvas.
   - Choose the uploaded template from the selection menu and click **Add**.
4. **Configure Connections & Services**:
   - Enable your Controller Services globally (specifically the Record Readers, Record Writers, and DBCPConnectionPools).
   - Configure the **ListFile / FetchFile** processor input directories to point to your `OUTPUT_DIR` (e.g., `D:\nifi_input\data`).
   - Configure **QueryDatabaseTableRecord** with your specific DB credentials, Table Name, and state tracking fields.
   - Configure **PutHDFS**: Supply your Hadoop configuration resources (usually `/etc/hadoop/conf/core-site.xml`, `hdfs-site.xml`) and set an appropriate target HDFS directory.
5. Once all yellow warnings disappear, press **Start** on the Process Group.

## Usage and Basic Examples

### Running the Python Data Generator
Once NiFi is prepared and listening, start flooding the pipeline using the terminal:

```bash
cd Generate_code
python generate.py
```

**Expected Output:**
```text
 Started! Generating INTRA-FILE duplicates every 3rd file...
 Created: batch_1715443200.json (4 records)
 Created: batch_1715443205.json (3 records)
 Added an INTRA-FILE DUPLICATE (ID: 1005) to batch_1715443210.json
...
```
The script reliably drops a batch every 5 seconds until manually terminated (`Ctrl + C`).

### End-to-End Execution
1. As Python creates files, `ListFile` and `FetchFile` will ingest them instantly.
2. `QueryDatabaseTableRecord` will routinely poll the target database for fresh inserts
3. The ingested dataset merges, passes through `ValidateRecord` and `DeduplicateRecord`—removing the intentional intra-file duplications shown locally.
4. `LogAttribute` triggers register standard output for operational observability.
5. Successfully transformed records successfully land in your Hadoop cluster via `PutHDFS`. Validate this by querying your cluster:
   `hdfs dfs -ls /your/target/hdfs/directory`
